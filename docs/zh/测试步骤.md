## 📋 测试清单

### 1. 打开浏览器
- ✅ 浏览器已自动打开
- URL: http://localhost:5118

### 2. 发送测试消息

请在浏览器的聊天界面中依次测试以下消息：

#### 测试 1: 提及特定 Agent
```
Hello @Sunny
```

**预期结果**:
- 控制台应显示:
  ```
  [DEBUG] Event type: AgentRunUpdateEvent
  [DEBUG] AgentRunUpdateEvent - ExecutorId: sunny, Update.Text: '...'
  [DEBUG] Event type: WorkflowOutputEvent
  ```
- 浏览器应显示 Sunny (☀️) 的回复

#### 测试 2: 不提及 Agent（默认）
```
Hello there!
```

**预期结果**:
- 应由默认的 Sunny agent 回复
- 控制台显示类似的 DEBUG 日志

#### 测试 3: 切换到不同的 Agent
```
@Techie tell me about AI
```

**预期结果**:
- 应由 Techie (🤖) 回复
- 控制台显示 ExecutorId: techie

#### 测试 4: 其他 Agents
```
@Artsy show me something beautiful
```
```
@Foodie what's for dinner?
```

## 🔍 观察要点

### 在控制台观察：
1. **是否有 [DEBUG] 日志输出？**
	- ✅ 如果有，说明事件监听正常
	- ❌ 如果没有，说明事件流有问题

2. **ExecutorId 是否正确？**
	- 应该看到 `sunny`, `techie`, `artsy`, `foodie` 等

3. **Update.Text 是否有内容？**
	- 应该看到实际的文本片段

### 在浏览器观察：
1. **是否显示 Agent 回复？**
	- ✅ 如果有，说明消息处理正常
	- ❌ 如果没有，检查浏览器控制台错误

2. **Agent 头像和名称是否正确？**
	- Sunny: ☀️
	- Techie: 🤖
	- Artsy: 🎨
	- Foodie: 🍜

3. **是否能正常切换 Agent？**
	- 使用 @ 提及应该能切换到对应的 agent

## ⚠️ 可能的问题

### 如果控制台没有 DEBUG 输出：
1. **检查 Azure OpenAI 配置**
	- 确保 `appsettings.Development.json` 包含正确的配置
	- 或者设置了环境变量

2. **查看是否有错误信息**
	- 在控制台查找红色的错误消息
	- 在浏览器开发者工具的 Console 中查看错误
